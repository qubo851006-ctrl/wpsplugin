# WPS AI 助手服务器版部署说明

## 目标形态

本项目已切换为内网集中部署模式：代理服务运行在服务器 `192.168.9.226`，同事电脑只需要在 WPS 中注册加载项地址，不需要本机启动代理，也不需要保存 API Key。

- 服务地址：`http://192.168.9.226:8765`
- 加载项地址：`http://192.168.9.226:8765/manifest.xml`
- 健康检查：`http://192.168.9.226:8765/api/status`

## 服务器需要做什么

1. 确认服务器 IP 是 `192.168.9.226`，且同事电脑能访问该地址。
2. 在服务器上进入项目目录：

```powershell
cd /d D:\claude\wps-ai-qa
```

3. 确认 `.env` 已配置真实接口信息：

```text
AIRCHINA_API_KEY=...
AIRCHINA_BASE_URL=...
ZHISHU_API_KEY=...
ZHISHU_BASE_URL=...
MODEL_CHAT=glm-5-outside
```

4. 安装 Python 依赖（只需一次）：

```powershell
pip install httpx python-dotenv urllib3
```

5. 启动服务：

方式 A：双击 `启动服务器.bat`。

方式 B：命令行运行：

```powershell
python proxy.py
```

启动后应看到：

```text
监听     : 0.0.0.0:8765
访问地址 : http://192.168.9.226:8765
```

6. 放行 Windows 防火墙 TCP `8765` 端口。

## 同事电脑需要做什么

1. 打开 WPS。
2. 进入“开发工具” -> “WPS 加载项”。
3. 添加加载项地址：

```text
http://192.168.9.226:8765/manifest.xml
```

4. 重启 WPS，功能区应出现“AI 助手”。

## 验证方法

在同事电脑浏览器打开：

```text
http://192.168.9.226:8765/api/status
```

如果返回类似下面内容，说明服务器和网络是通的：

```json
{"status":"ok","model":"glm-5-outside"}
```

## 注意事项

- `.env` 只放服务器，不要分发给同事。
- 同事电脑不要再使用本机 `127.0.0.1:8765` 的安装方式。
- 后续升级功能时，只需要更新服务器目录下的代码并重启 `proxy.py`。
